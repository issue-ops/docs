## GitHub APIs

<DoDontContainer>
  <Do indented>Use GitHub Apps for accessing organization-level APIs</Do>
  <Dont indented>Use personal access tokens</Dont>
</DoDontContainer>

## Sensitive information

<DoDontContainer>
  <Do indented>
    Use issue forms inputs that accept references to sensitive information in
    secure locations
  </Do>
  <Dont indented>Accept sensitive information directly in issues</Dont>
</DoDontContainer>

## Validation

<DoDontContainer>
  <Do indented>
    Validate issue and comment text at every step in the IssueOps workflow
  </Do>
  <Dont indented>
    Rely on labels to determine if an issue has been validated or approved
  </Dont>
</DoDontContainer>
